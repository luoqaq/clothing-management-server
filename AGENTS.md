# AGENTS 工作规范

## 作用范围
- 本文件对整个仓库生效。

## 环境基线
- 本仓库默认使用 `npm` 脚本执行常规流程。
- 后端默认端口是 `3000`。
- 数据库连接配置必须来自 `.env`，默认连接真实数据库。
- 除非用户明确要求，不切换为 mock DB。

## 常用命令
- 安装依赖：`npm install`（必要时可用 `bun install`）
- 开发模式：`npm run dev`
- 启动：`npm run start`
- 构建：`npm run build`
- 健康检查：`curl http://127.0.0.1:3000/health`

## 必要会话流程
1. 每次新会话开始时，先读取 `PROJECT_MEMORY.md`。
2. 仅当本次会话产生了值得复用的高价值信息时，才在会话结束前更新 `PROJECT_MEMORY.md`。
3. 记忆更新采用“会话结束统一总结后再写入”的方式，不在处理中频繁追加零散记录。
4. 收尾回复固定包含三项：
   - 改了什么
   - 如何验证
   - 剩余风险或待办

## 变更边界
- 仅修改与当前用户需求直接相关的文件。
- 未经用户要求，不做顺手重构。
- 如果发现与当前任务冲突的意外本地改动，先暂停并确认再继续。

## 验证规则
- 代码修改后，至少执行一项相关验证（`build`、测试或定向运行验证）。
- 如无法验证，必须明确说明未验证项。

## 安全规则
- 回复中不得输出密钥、密码、令牌等敏感信息。
- 后端 `.env` 严禁提交到版本库。
- 未经明确要求，不修改 `.env` 文件。

## Git 规则
- 提交信息清晰可读，优先使用 Conventional Commit 风格。
- 未经明确要求，不执行 `commit --amend`。
- 未经明确要求，不执行强推（force push）。

## 记忆更新规则
- 内容保持简洁、事实化。
- 仅记录高价值信息：环境或配置变化、运行与验证结果、问题与根因、关键决策和约束。
- 只有当改动对后续会话、联调、部署、排障或方案延续有明显帮助时，才更新记忆。
- 记忆更新时机固定为：本次会话工作基本结束后，统一总结并写入。
- 更新 `PROJECT_MEMORY.md` 时使用固定模板：会话日期、变更内容、验证结果、遗留问题或风险。
- 避免重复陈旧或低价值信息。
